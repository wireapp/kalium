The message doesn't contain an actual content, but rather instructions on how to read an external message.

This message content is used if original message results in large payload, that would not be accepted by backend. Regular messages over Proteus are encrypted multiple times (per recipient) and in case of multiple participants even quite small message can generate huge payload. In that case we want to encrypt original message with symmetric encryption and only send a key to all participants.
